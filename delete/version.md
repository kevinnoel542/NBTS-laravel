# Version Plan

This document tracks planned product versions for the National Blood Donation and Blood Bank Management Platform.

## Version 0.1 - Core Foundation

Goal: establish the system structure for admins, staff, donors, centers, and secure access.

Scope:
- Roles and permissions
- User accounts
- Donor profiles
- Blood centers
- Center staff assignment
- Authentication
- Filament access control

## Version 0.2 - Mobile API Foundation

Goal: support the Flutter donor app with secure API endpoints.

Scope:
- Donor registration
- Donor login/logout
- Profile API
- Donor card API foundation
- Blood center API
- Campaign API

## Version 0.3 - Appointment and Donation Operations

Goal: allow donors to book appointments and staff to record donations.

Scope:
- Appointment booking
- Appointment status management
- Walk-in donation support
- Donor search
- QR/donor ID lookup
- Donation recording
- Blood group verification

## Version 0.4 - Eligibility, Donor Card, and Loyalty

Goal: improve donor safety, verification speed, and retention.

Scope:
- Eligibility engine
- Deferrals
- Next eligible donation date
- Digital donor ID
- QR code generation
- Badges
- Rewards
- Leaderboards

## Version 0.5 - Inventory and Notifications

Goal: track blood supply and notify the right people at the right time.

Scope:
- Blood unit lifecycle
- Inventory totals
- Manual inventory adjustments
- Expiry handling
- Low stock alerts
- Emergency campaigns
- Push/SMS/email notification history

## Version 1.0 - Production Readiness

Goal: prepare the platform for real NBTS operations.

Scope:
- Public website completion
- Reports and analytics
- API tests
- Role permission tests
- Donation workflow tests
- Inventory workflow tests
- Deployment setup
- Production environment hardening
